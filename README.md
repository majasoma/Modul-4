## Reflection
Documentation, diagrams, and a clear project structure help a team build shared understanding of how the code works. Automated tests make it easier to catch bugs early and give confidence when making changes. Version control with Git helps track progress and makes collaboration safer. Pair programming, mob programming, and peer reviews help more developers understand the codebase, not just one person. These practices support shared code ownership because knowledge is shared across the team instead of being kept by only one developer.
